# nekode

A tamagotchi-style desktop companion that shows you what Claude Code is doing. Built with an ESP32-S3, a tiny OLED display, RGB LEDs, and a microphone.

Even with a second monitor showing Claude Code, it's easy to miss when it needs input or finishes a task. nekode sits on your desk and gives you an at-a-glance status — a cute animated kitty that reacts to Claude's state in real-time.

## How it works

Claude Code hooks send serial commands to the ESP32 whenever the agent's state changes. The kitty animates accordingly and the RGB LED strip lights up while Claude is active.

```
Claude Code hook → set_status.sh → USB serial → ESP32 → OLED + LEDs
```

## Hardware

| Part | Details |
|------|---------|
| MCU | ESP32-S3-DevKitC-1 |
| Display | SSD1306 128x64 OLED (I2C) |
| LEDs | 20x APA106 RGB strip |
| Mic | INMP441 I2S microphone |

## Serial commands

Send newline-terminated commands over USB serial at 115200 baud:

```
thinking, typing, running, reading, searching, planning,
browsing, spawning, waiting, done, error, sleep, boot,
herding, compacting
```

## Build

Requires [PlatformIO](https://platformio.org/).

```
pio run              # build
pio run -t upload    # flash
pio device monitor   # serial monitor
```

## Project structure

```
src/
  config.h         — pin definitions, timing constants
  main.cpp         — setup/loop, serial command dispatcher
  oled.h/.cpp      — OLED display: neko animation + spectrum analyser
  led_strip.h/.cpp — RGB LED strip: rainbow animation via RMT
  mic.h/.cpp       — INMP441 microphone: I2S RMS volume
include/
  *.h              — animation frames (generated by gif_to_header.py)
gif_to_header.py   — converts GIF → C header with PROGMEM frame arrays
set_status.sh      — sends serial commands to ESP32
```
